# 04. SQL и моделирование данных

## Цель модуля
Освежить базовые знания SQL и моделирования данных, необходимые System / Business Analyst для анализа требований и общения с разработчиками.

---

## 1. Роль SQL для аналитика
Аналитик использует SQL для:
- анализа данных
- проверки гипотез
- поиска проблем
- валидации логики

Аналитик не обязан писать сложные оптимизированные запросы,
но должен **понимать структуру данных**.

---

## 2. Базовые SQL операции

SELECT — выборка данных  
INSERT — добавление  
UPDATE — обновление  
DELETE — удаление  

---

## 3. JOIN и связи таблиц

Типы связей:
- one-to-one
- one-to-many
- many-to-many

Типы JOIN:
- INNER JOIN
- LEFT JOIN

---

## 4. Нормализация БД

1NF — атомарные значения  
2NF — нет частичных зависимостей  
3NF — нет транзитивных зависимостей  

Цель — уменьшить дублирование и повысить целостность данных.
