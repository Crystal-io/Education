# 02. Web and Mobile Architecture

## Цель модуля
Сформировать практическое понимание архитектуры web и mobile приложений, достаточное для уверенных ответов на интервью System / Business Analyst.

Модуль ориентирован на объяснение архитектуры простым, прикладным языком — без углубления в уровень Solution Architect.

---

## 1. Общая архитектурная картина

Современные продуктовые системы, как правило, строятся по client-server модели:

Client (Web / Mobile)
→ API Layer
→ Backend Services
→ Databases / External Systems

Задача аналитика — понимать:
- где возникает бизнес-логика
- где происходит валидация
- где хранятся данные
- как компоненты взаимодействуют между собой

---

## 2. Web-приложения (Frontend)

Типовая структура:
- Browser
- SPA (React / Angular / Vue)
- HTTP / HTTPS

Роль frontend:
- отображение данных
- сбор пользовательского ввода
- вызов API
- базовая валидация (не бизнес-логика)

Важно:
Frontend не является источником истины.

---

## 3. Mobile-приложения (iOS / Android)

Типовая схема:
- iOS / Android App
- REST API
- Backend Services

Особенности mobile:
- нестабильное соединение
- offline-сценарии
- повышенное внимание к ошибкам и retry

---

## 4. Backend

Backend отвечает за:
- бизнес-логику
- валидацию данных
- работу с БД
- интеграции с внешними сервисами

Для аналитика важно:
- понимать границы сервисов
- уметь читать и описывать API-контракты
- учитывать non-functional требования

---

## 5. API как точка взаимодействия

API — контракт между клиентом и сервером.

Типовые характеристики:
- REST
- JSON
- Stateless
- Versioning

Аналитик описывает:
- endpoints
- request / response
- error cases
- ограничения

---

## 6. Архитектура глазами аналитика

Аналитик НЕ проектирует архитектуру, но:
- понимает её логику
- выявляет риски
- задаёт правильные вопросы
- помогает команде договориться

Примеры вопросов:
- где будет происходить валидация?
- как обрабатываются ошибки?
- что будет при timeout?
- как система масштабируется?

---

## 7. Типовые вопросы на интервью

Как устроено web-приложение?
Как mobile клиент взаимодействует с backend?
В чём разница между frontend и backend?
Что такое API и зачем оно нужно?
